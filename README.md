# Описание алгоритма 
Алгоритм считает аномалией черезмерно высокую дневную активность конкретного респондента по определенному бренду и категории в определенный день по сравнению с типичной активностью респондентов по этим группам; малые значения активности игнорируются и не считаются аномалией

Для оценки находится robust z-score, который не "раздувается" от экстремальных выбросов

Пороги выбираются лвумя методами:
1. Статистический порог - score > 3.5
2. Динамический порог - daily_ots > (median * 0.4 ) + 5

Это значит, что значение должно не только статистически выбиваться, но и быть существенно больше медианы группы

# Обоснование выбора метода
1. Устойчивость - использование медианы и MAD вместо среднего и stdev позволяет избежать искажение статистики по группе большими выбросами
2. Производительность и детерминизм - алгоритм использует векторизированные операции pandas, не содержит случайных значений
3. Интерпретируемость - для каждой аномалии указана причина

# Установка и запуск 
В файле requirements.txt указаны приблизительные версии библиотек, которые необходимо установить (приблизительные потому что я работаю на NixOS и к счастью не использую pip поэтому не могу посмотреть точные версии)

Для запуска:
```
pip install -r requirements.txt
python solution_SHAD-111_Kutaev.py
```
